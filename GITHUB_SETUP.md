## 8. Create Release Tags

Create semantic versioned releases:

```bash
git tag -a v1.0.0 -m "Initial release: 33 agents, 80+ skills"
git push origin v1.0.0
```

Then create a GitHub Release with:
- Release notes
- Highlight major features
- Link to CHANGELOG
